# 局域网文件传输系统开发文档

## 1. 项目概述

### 1.1 项目名称

局域网文件传输系统

### 1.2 项目目标

本项目基于 C# 语言开发，实现局域网环境下多台电脑之间的文件传输。系统采用 C/S 架构，客户端通过 TCP 连接服务端，服务端负责接收文件、保存文件、发送文件以及管理文件信息。

### 1.3 核心技术

| 技术 | 用途 |
|---|---|
| C# | 项目开发语言 |
| WinForms | 客户端图形化界面 |
| Console | 服务端控制台程序 |
| TcpClient | 客户端 TCP 通信 |
| TcpListener | 服务端 TCP 监听 |
| NetworkStream | 网络数据流读写 |
| Entity Framework Core | 数据库访问 |
| SQLite | 本地轻量级数据库 |
| ZIP 压缩 | 文件夹传输 |

## 2. 系统总体设计

### 2.1 系统架构

系统采用三层结构：

```text
客户端 Client
    ↓ TCP 通信
服务端 Server
    ↓ EF Core
数据库 SQLite
    ↓
服务端本地文件目录
```

### 2.2 架构说明

| 模块 | 说明 |
|---|---|
| 客户端 | 提供图形化界面，负责连接服务端、上传、下载、查看文件列表 |
| 服务端 | 监听客户端连接，处理上传、下载、数据库记录等业务 |
| 数据库 | 保存用户信息、文件信息、传输记录 |
| 文件存储目录 | 保存实际上传的文件内容 |

### 2.3 项目结构

```text
LanFileTransfer
├── LanFileTransfer.Client       客户端项目
├── LanFileTransfer.Server       服务端项目
├── LanFileTransfer.Common       公共类库
└── LanFileTransfer.Data         数据访问类库
```

### 2.4 项目引用关系

```text
Client  → Common
Server  → Common
Server  → Data
Data    → Common，可选
```

说明：

- `Common` 存放客户端和服务端都要使用的公共内容。
- `Data` 存放数据库实体和 EF Core 数据库上下文。
- 客户端不直接访问数据库。
- 所有数据库操作都由服务端完成。

## 4. 通信设计

### 4.1 通信方式

客户端和服务端使用 TCP 通信。

```text
客户端 TcpClient
    ↓ NetworkStream
服务端 TcpListener
```

### 4.2 通信原则

- TCP 是字节流，不能假设一次读取就是完整消息。
- 必须设计统一消息格式。
- 文件内容需要分块传输。
- 大文件不能一次性读入内存。

### 4.3 消息格式设计

推荐消息格式：

```text
消息头长度 + 消息头内容 + 文件内容
```

| 部分 | 说明 |
|---|---|
| 消息头长度 | 表示消息头占多少字节 |
| 消息头内容 | 保存消息类型、文件名、大小、用户等信息 |
| 文件内容 | 上传或下载时传输的二进制数据 |

### 4.4 消息类型设计

| 消息类型 | 方向 | 作用 |
|---|---|---|
| LoginRequest | 客户端 → 服务端 | 登录请求 |
| LoginResponse | 服务端 → 客户端 | 登录结果 |
| RegisterRequest | 客户端 → 服务端 | 注册请求 |
| RegisterResponse | 服务端 → 客户端 | 注册结果 |
| UploadRequest | 客户端 → 服务端 | 上传请求 |
| UploadResponse | 服务端 → 客户端 | 上传结果 |
| FileListRequest | 客户端 → 服务端 | 获取文件列表 |
| FileListResponse | 服务端 → 客户端 | 返回文件列表 |
| DownloadRequest | 客户端 → 服务端 | 下载请求 |
| DownloadResponse | 服务端 → 客户端 | 下载结果 |
| TransferRecordRequest | 客户端 → 服务端 | 获取传输记录 |
| TransferRecordResponse | 服务端 → 客户端 | 返回传输记录 |
| ErrorResponse | 服务端 → 客户端 | 错误提示 |

### 4.5 上传请求字段

| 字段 | 说明 |
|---|---|
| UserId | 上传用户编号 |
| ResourceName | 文件或文件夹名称 |
| ResourceType | 资源类型 |
| FileSize | 文件大小 |
| FileHash | 文件校验值，可选 |
| OriginalFileName | 原始文件名 |
| Extension | 文件扩展名 |

### 4.6 资源类型设计

| 类型 | 说明 |
|---|---|
| File | 普通单文件 |
| FolderZip | 文件夹压缩包 |
| MultiFileZip | 多文件压缩包 |

## 5. 文件传输设计

### 5.1 单文件上传流程

```text
客户端选择文件
↓
读取文件名和文件大小
↓
发送上传请求
↓
服务端创建临时文件
↓
客户端分块发送文件内容
↓
服务端分块写入临时文件
↓
传输完成后校验文件大小
↓
临时文件移动到正式目录
↓
数据库保存文件信息
↓
数据库保存上传记录
↓
服务端返回上传成功
```

### 5.2 单文件下载流程

```text
客户端请求文件列表
↓
服务端查询数据库
↓
客户端选择文件
↓
发送下载请求
↓
服务端根据文件 ID 查询文件路径
↓
服务端分块读取本地文件
↓
服务端通过 TCP 发送文件内容
↓
客户端分块接收并保存
↓
数据库保存下载记录
```

### 5.3 文件夹传输设计

文件夹不直接传输，采用 ZIP 压缩方式。

```text
客户端选择文件夹
↓
客户端将文件夹压缩为 ZIP
↓
按普通文件方式上传 ZIP
↓
服务端保存 ZIP 文件
↓
数据库记录资源类型为 FolderZip
↓
其他客户端下载 ZIP
↓
客户端解压还原文件夹结构
```

### 5.4 分块传输设计

建议分块大小：

```text
64KB 或 8192 字节
```

设计要求：

- 上传时循环读取本地文件。
- 下载时循环读取服务端文件。
- 每传输一块更新一次进度。
- 根据已传输字节数和总大小计算百分比。

## 6. 文件存储设计

### 6.1 存储原则

- 文件内容保存到服务端本地磁盘。
- 数据库只保存文件元数据。
- 上传中的文件先保存到临时目录。
- 上传成功后再移动到正式目录。
- 服务端保存文件名使用唯一名称，避免重名。

### 6.2 目录结构

```text
ServerStorage
├── files
│   ├── 2026-06-19
│   └── 2026-06-20
└── temp
```

### 6.3 文件命名规则

| 名称 | 说明 |
|---|---|
| OriginalFileName | 用户上传时的原始文件名 |
| StoredFileName | 服务端保存时生成的唯一文件名 |
| FilePath | 服务端实际保存路径 |

服务端保存文件名建议：

```text
GUID + 原文件扩展名
```

示例：

```text
原始文件名：课程设计.docx
服务端文件名：3f81a2e9c8a94b1e.docx
```

## 9. 服务端设计

### 9.1 服务端职责

- 启动 TCP 监听。
- 接收客户端连接。
- 为每个客户端创建独立处理任务。
- 根据消息类型处理不同请求。
- 操作数据库。
- 保存和读取文件。
- 输出运行日志。

### 9.2 服务端主要模块

| 模块 | 作用 |
|---|---|
| TcpServer | 启动监听，接收连接 |
| ClientHandler | 处理单个客户端请求 |
| UserService | 处理登录和注册 |
| FileService | 处理文件保存、读取、删除 |
| TransferService | 保存传输记录 |
| DatabaseService | 封装数据库访问 |
| ServerLogger | 输出服务端日志 |

## 10. 简单错误提示设计

本项目是课程大作业，不单独设计复杂异常处理框架，只在功能附近保留必要提示和日志。

| 异常情况 | 处理方式 |
|---|---|
| 客户端连接失败 | 客户端提示服务器不可用 |
| 上传中断 | 删除临时文件，记录失败日志 |
| 下载中断 | 客户端提示下载失败 |
| 文件不存在 | 服务端返回错误响应 |
| 数据库写入失败 | 返回失败结果并输出服务端日志 |
| 文件名重复 | 使用 GUID 文件名避免冲突 |
| ZIP 压缩失败 | 客户端提示文件夹上传失败 |
| ZIP 解压失败 | 客户端提示手动解压 |
