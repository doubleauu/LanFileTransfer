## 第 2 页：项目背景与目标

- 局域网内经常需要在多台电脑之间传输文件。
- U 盘传输不方便，聊天软件或网盘依赖外部平台。
- 本项目使用 C# 网络编程实现局域网文件传输系统。
- 系统重点练习客户端、服务端、TCP 通信、文件流和数据库保存。
- 在非同一局域网环境下，也可以配合皎月连进行内网穿透连接测试。

主要目标：

- 客户端可以连接服务端。
- 用户可以注册和登录。
- 支持单文件上传和下载。
- 支持文件夹压缩为 ZIP 后上传和下载。
- 支持文件列表、传输记录和进度显示。
- 支持局域网 IP 连接，也支持通过皎月连虚拟 IP 或端口映射进行跨网络访问。

## 第 3 页：系统总体架构

本项目采用 C/S 架构：

- Client：WinForms 客户端，负责界面显示和用户操作。
- Server：Console 服务端，负责监听连接、处理请求和保存文件。
- Common：公共类库，负责 TCP 命令协议和公共枚举。
- Data：数据访问类库，负责 EF Core 和 SQLite 数据库。

数据流向结构图：

```text
客户端界面 -> TCP 命令协议 -> 服务端业务处理 -> SQLite / ServerStorage
```

连接方式：

```text
局域网访问：服务端IPv4:5000
皎月连组网模式：虚拟IP:5000
皎月连映射模式：127.0.0.1:远程端口
```

## 第 7 页：TCP 通信协议设计

为了解决 TCP 连接无边界问题，本项目使用特定分隔符，普通命令字符串作为业务协议，另外使用了转义字符 ‘//’ 解决命令中额外分隔符可能带来的影响

```text
LoginRequest|用户名|密码
RegisterRequest|用户名|密码
UploadRequest|用户Id|资源类型|文件大小|原文件名|扩展名
DownloadRequest|用户Id|文件Id
FileListRequest|用户Id
TransferRecordRequest|用户Id
```

命令发送格式：

```text
4 字节命令长度 + UTF-8 命令字符串
```

设计原因：

- TCP 是字节流，不能保证一次读取就是一条完整命令。
- 4 字节长度前缀足够大，可以明确知道后面命令字符串有多长。

## 第 8 页：单文件上传与下载流程

单文件上传：

1. 用户在客户端选择本地文件。
3. 客户端读取文件名、扩展名和文件大小，并发送上传命令。
4. 客户端使用 `FileStream` 分块读取文件，并通过 `NetworkStream` 分块发送文件内容。
6. 服务端先写入临时文件，之后校验接收大小，校验成功后移动到正式保存目录。
9. 服务端写入文件记录和上传记录。
10. 服务端给客户端返回上传结果。

单文件下载：

1. 用户在文件列表中选择文件。
2. 客户端发送下载命令。
3. 服务端根据文件 ID 查询数据库并检查文件。
4. 服务端返回文件基本信息。
5. 服务端通过 `FileStream` 和 `NetworkStream` 分块发送文件内容。
6. 客户端分块接收文件，写入本地并更新进度条。
7. 服务端保存下载记录。

## 第 9 页：文件夹 ZIP 传输设计

背景引入：

- 单个文件可以直接通过文件流上传和下载。
- 文件夹本身包含多个文件和子目录，不能像普通文件一样直接作为一个连续文件流发送。
- 如果逐个文件传输，需要额外设计目录结构、文件数量、相对路径等协议，复杂度会明显增加。
- 因此本项目采用“先压缩为 ZIP，再按普通文件传输”的方式实现文件夹传输。

文件夹传输设计：

- 上传文件夹前，客户端先把文件夹压缩为 ZIP。
- ZIP 文件按普通文件上传。
- 服务端保存 ZIP 文件，并记录资源类型为 `FolderZip`。
- 下载时客户端接收 ZIP 文件。
- 下载完成后客户端可以选择是否解压。

设计优点：

- 复用已有的单文件上传和下载流程。
- 保留文件夹内部目录结构。
- 实现方式简单，用户不需要手动提前压缩或下载后手动处理复杂目录。

## 第 10 页：数据库设计

数据库使用 EF Core + SQLite。

C# 与数据库的实现路径：

- C# 代码不直接手写 SQL，而是通过 EF Core 操作实体类和 `DbContext`。
- 查询数据时，业务代码编写 LINQ 表达式。
- EF Core 根据实体类和表结构映射关系，将 LINQ 转换为 SQL 语句。
- SQLite 执行 SQL 后返回结果，EF Core 再把查询结果转换成 C# 对象。
- 新增或修改数据时，先修改实体对象，再调用 `SaveChanges` 主动写入数据库。

主要数据表：

| 表名 | 作用 |
| --- | --- |
| `User` | 保存用户账号和密码哈希 |
| `FileRecord` | 保存文件名、大小、保存路径、资源类型和上传时间 |
| `TransferRecord` | 保存上传、下载记录和传输状态 |

## 第 12 页：项目总结

本项目主要完成了：

- WinForms 客户端界面。
- TCP 客户端和服务端通信。
- 自定义命令协议。
- 单文件和文件夹传输。
- 文件分块上传和下载。
- EF Core + SQLite 数据保存。
- 文件列表和传输记录查询。

项目收获：

- 理解了 C/S 架构的基本工作方式。
- 掌握了 TCP 网络通信的基本流程。
- 理解了 TCP 字节流中的粘包和拆包问题。
- 练习了文件流读写和数据库保存。
- 提升了小组分工、联调和答辩展示能力。

后续优化：

- 大文件传输速度优化：在现有分块传输基础上，改为“分片并行传输”。客户端把大文件按固定大小切成多个片段，例如每片 4MB，开启 3 到 4 个 TCP 连接并行上传；服务端按片段编号或偏移量写入临时文件，全部片段完成后再校验文件大小并合并为正式文件。同时把缓冲区调整到 256KB 或 1MB，并降低进度条刷新频率，减少 UI 更新对传输速度的影响。
- UI 美化：进一步整理按钮、表格和日志区域布局，增加更清晰的状态提示，让演示界面更加直观。
