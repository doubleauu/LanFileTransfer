# 内网穿透说明书

本文档用于说明本项目如何通过皎月连进行跨网络访问测试。项目服务端默认监听 `5000` 端口，客户端通过 TCP 连接服务端完成登录、文件上传、文件下载、文件列表查询和传输记录查询。

## 一、为什么需要内网穿透

本项目是 C/S 架构：

```text
WinForms 客户端 -> TCP 连接 -> Console 服务端
```

在同一个局域网中，客户端可以直接连接服务端电脑的局域网 IPv4 地址，例如：

```text
192.168.1.20:5000
```

但是如果客户端和服务端不在同一个局域网中，例如服务端在宿舍 WiFi，客户端在手机热点或其他网络中，客户端通常不能直接访问 `192.168.x.x` 这样的内网地址。

内网穿透的作用就是在两端之间建立一条转发通道，让外网客户端的请求可以到达内网中的服务端程序。

## 二、本项目的服务端端口

服务端默认监听端口是：

```text
5000
```

服务端入口文件中使用 `TcpListener` 启动监听，并且监听地址是 `IPAddress.Any`，表示服务端可以接收来自本机、局域网 IP、虚拟网卡 IP 的连接。

因此本项目可以支持三种访问方式：

```text
局域网访问：服务端IPv4:5000
皎月连组网模式：虚拟IP:5000
皎月连映射模式：127.0.0.1:远程端口
```

## 三、推荐方案：组网模式

实际测试中，本项目使用皎月连组网模式更稳定。组网模式可以理解为：皎月连在客户端和服务端之间创建一个虚拟局域网，客户端通过皎月连分配的虚拟 IP 访问服务端。

### 1. 服务端电脑操作

服务端电脑就是运行本项目 `LanFileTransfer.Server` 的电脑。

1. 启动 C# 服务端程序。
2. 确认服务端正在监听 `5000` 端口。
3. 打开皎月连并登录。
4. 左侧选择“服务端”。
5. 开启“组网模式”。
6. 设置访问密码。
7. 点击“开启服务”。

### 2. 客户端电脑操作

客户端电脑就是运行本项目 `LanFileTransfer.Client` 的电脑。

1. 打开皎月连并登录。
2. 左侧选择“客户端”。
3. 在在线主机列表中选择服务端电脑。
4. 点击“连接主机”。
5. 连接成功后，查看皎月连显示的虚拟局域网 IP，例如：

```text
10.6.22.1
```

6. 打开本项目客户端程序。
7. 客户端连接信息填写：

```text
服务器地址：10.6.22.1
服务器端口：5000
```

这里的 `10.6.22.1` 只是示例，实际要填写皎月连界面显示的虚拟 IP。

### 3. 组网模式访问流程

```text
外网 C# 客户端
    |
    | 连接 虚拟IP:5000
    v
皎月连虚拟局域网
    |
    v
服务端电脑
    |
    | 转交给 5000 端口
    v
C# TcpListener 服务端程序
```

组网模式下，客户端连接方式更接近普通局域网访问，只是局域网 IP 换成了皎月连提供的虚拟 IP。

## 四、备选方案：映射模式

映射模式不是创建完整虚拟局域网，而是只转发指定端口。它的作用是把客户端本机的一个端口转发到服务端电脑的真实端口。

### 1. 服务端电脑配置

1. 启动 C# 服务端程序。
2. 打开皎月连并登录。
3. 左侧选择“服务端”。
4. 关闭“组网模式”。
5. 点击“添加端口映射”。
6. 添加 TCP 映射：

```text
协议：TCP
本地地址：127.0.0.1
本地端口：5000
远程端口：15000
```

这里建议远程端口使用 `15000`，方便和服务端真实端口 `5000` 区分。

### 2. 客户端电脑配置

1. 打开皎月连并登录。
2. 左侧选择“客户端”。
3. 选择服务端主机。
4. 点击“连接主机”。
5. 打开本项目客户端程序。
6. 客户端连接信息填写：

```text
服务器地址：127.0.0.1
服务器端口：15000
```

### 3. 映射模式访问流程

```text
外网 C# 客户端
    |
    | 连接 127.0.0.1:15000
    v
客户端电脑上的皎月连
    |
    | 皎月连转发通道
    v
服务端电脑上的皎月连
    |
    | 转发到 127.0.0.1:5000
    v
C# TcpListener 服务端程序
```

这里要注意：客户端填写的 `127.0.0.1` 不是服务端电脑地址，而是客户端电脑上皎月连创建的本地转发入口。真正负责找到服务端的是皎月连。

## 五、组网模式和映射模式的区别

| 对比项 | 组网模式 | 映射模式 |
| --- | --- | --- |
| 本质 | 创建虚拟局域网 | 转发指定端口 |
| 客户端填写 | 虚拟IP:5000 | 127.0.0.1:远程端口 |
| 适合场景 | 多端口、多服务、模拟局域网 | 单个 TCP 服务 |
| 本项目推荐 | 推荐 | 作为备选 |
| 答辩表达 | 更接近局域网访问 | 体现端口转发原理 |

本项目只需要访问一个 TCP 服务端口，所以理论上映射模式也可以使用。但实际测试中，映射模式在打开客户端时可能导致皎月连断开，因此最终演示建议使用组网模式。

## 六、答辩讲解话术

可以按下面这段讲：

> 我的项目原本是局域网文件传输系统，客户端通过 `TcpClient` 连接服务端的 `TcpListener`。在同一局域网中，客户端可以直接连接服务端电脑的 IPv4 地址和 `5000` 端口。但是在不同网络环境下，普通内网地址无法直接访问，所以我使用皎月连进行内网穿透测试。
>
> 皎月连不参与项目的业务逻辑，只负责网络层转发。我的服务端程序仍然监听 `5000` 端口，登录、上传、下载、文件列表等功能仍然使用项目原来的 TCP 通信协议。
>
> 我测试了皎月连的组网模式和映射模式。映射模式是端口转发，客户端访问本机的转发端口，再由皎月连转发到服务端的 `5000` 端口。组网模式是创建虚拟局域网，客户端通过皎月连分配的虚拟 IP 直接访问服务端 `5000` 端口。实际测试中组网模式更稳定，所以最终演示采用组网模式。

## 七、老师可能问的问题

### 1. 内网穿透会不会改变你的文件传输协议？

不会。内网穿透只改变网络访问路径，不改变应用层协议。项目仍然使用 `TcpClient`、`TcpListener` 和 `NetworkStream` 传输数据。

### 2. 为什么组网模式下客户端填虚拟 IP，而不是 `127.0.0.1`？

因为组网模式创建的是虚拟局域网，服务端会有一个虚拟网络地址。客户端访问这个虚拟 IP，就相当于在同一个虚拟局域网中访问服务端。

### 3. 为什么映射模式下客户端填 `127.0.0.1`？

因为映射模式会在客户端本机创建一个本地转发入口。客户端连接的是自己电脑上的皎月连入口，皎月连再把数据转发到服务端。

### 4. 为什么最终选择组网模式？

因为本项目实测中组网模式连接稳定，可以直接访问服务端 `5000` 端口，演示效果更可靠。映射模式虽然理论上适合单端口服务，但实际环境中可能受到 P2P、防火墙或软件拦截影响。
